Image Processing

Posted on 2020-02-26 | In Machine Learning

Recently, I have read some open-source codes about the Image Processing. The main modules used are ‘opencv’, ‘numpy’,’matplotlib’.

Then, the image is read into the file, and the type of which is np.ndarray. By the way, I have learnt that the image is stored in the computer in the form of matrix, where each pixel is consist of RGB 3 channels, and the value of each channel is between 0-255 as there are 8 bits in each channel. Therefore, through different combination of colours, we can see the colourful images. In addition, the shape of the image read is 506 * 675 * 3, meaning that the image is 506px in height, 675px in width and has 3 colour channels.

After this, if we want to see the image in the notebook, we can use plt.imshow() to show the image, and convert it from BGR to RGB colours. Also, we can show the grayscale image by converting BGR to GRAY.

BGR:

Grayscale:

After the pre-processing, the slicing can be used to choose the part you want.

Just like the slicing of list or string in Python, [start:stop] can be used to slice. Hence, the height and width can be chosen. Furthermore, yellow rectangles can be drawn to locate the chosen area. Note that the rectangle is drawn, and the parameter is the coordinate of the diagonal.

Additionally, binary image can also be shown, meaning that value greater than the threshold will be changed to 255, and that smaller than the threshold will be changed to 0, so that there are only two colours in the matrix, black and white, and 0 stands for black, 255 stands for white. The higher the threshold. the simpler the processed image, but there may be some loss in comparison with the original one. If the threshold is low, the processed image may be closer to the raw image, but there may be some noise in it.

What is more, the edge detection can be achieved using Laplace operator. By the way, I have learnt the simplest meaning of convolution. Just put matrix B on the matrix A, and multiply with the corresponding element. For example, if B is on the left upper corner of A, than the calculation should be 3 * 0 + 3 * 1 + 2 * 2 + 2 * 0 + 2 * 0 + 0 * 1 + 3 * 0 + 1 * 1 + 2 * 2 = 3 + 4 +1 +4 = 12, which is written in the left upper corner of matrix A * B.

In conclusion, I have learnt the following briefly:

  1. Images are stored as matrix.
  2. Different values in the matrix (0-255) represents the colour of R/G/B
  3. To process the image is actually to change the pixel represented by the matrix. It is an algebraic calculation
  4. 2D Convolution is calculated by multiplying corresponding elements
  5. Binary value can show the edge of an object
  6. Images have many types, such as RGB and Grayscale. Grayscale image is often used to process, and then convert it into the RGB or BGR form.
