Yesterday, I have added some functions to the previous codes.

  1. Auto-capture function
  2. Single capture function

Single-capture

  • If there is moisture data being read from Arduino, the moisture data will be written into the saved file name
  • The file name is composed of the current time and the current moisture
  • If moisture data is not successfully read to Python, ‘no data’ will be written in the filename
  • cv2.imwrite() is used to save the image.
  • try, except is used to make the code more robust. Otherwise, if there is a mistake in somewhere, the whole codes will collapse

Auto-capture

First of all, we set up a thread on the background:

The main idea is that once the autoCapturing is set to be True, the program will capture the image in a given interval of time. The status of autoCapturing is determined by whether you click the button:

Once the button is clicked, it will link to capture_auto_switch:

  • The initial value of autoCapturing is set to be False, so code will enter the first condition if the button is clicked
  • Once the button is clicked at the 1st time, the label will show ‘running’, and text on the button will change to ‘finish capture’. Also, the status of autoCapturing becomes True, so the autoCapturing threading on the background will run.
  • The next time you click the button, the codes will enter the second condition which is the parts in ‘else’, which is used to stop autoCapturing, and the value of which is set to False
  • This process repeats when you click the button.
