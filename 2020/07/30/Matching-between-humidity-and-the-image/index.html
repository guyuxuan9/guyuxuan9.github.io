Matching between humidity and the image

Posted on 2020-07-30 | In Dialogue with Plants |

Yesterday, I have written some codes to match the humidity data and the image by machine learning. In addition, I have adjusted the position of two cameras to keep capturing the images.

Machine Learning

First of all, I have transferred the images stored in the Raspberry Pi to my PC. I have kept 2852 images from Cam2, the magnifying camera, and 1984 images from Cam1, the normal camera.

Note that tqdm module is used to show the progress.

Get humidity data

This is used to store the humidity data into a .csv file. Note that the humidity data is changed into float type from integer type and then normalised, so that it fits into the required format of input of the neural network, which is a number between 0-1.

Read the images

imgs is an numpy array that is used to store the image matrix. Here, 4 dimensions are used instead of 3, because it will be easier to input all the pictures at once into the neural network. For example, if the final array is (1000,112,112,3), it means that 1000 images with sizes 112 * 112 are input as a whole to the network. The red bar shows the progress while reading the data. Initially, CPU can achieve 60-70 iterations per second, but as time passes by, it can only achieve 3-4 iterations per second, because the RAM is occupied by many images, and maybe virtual RAM from hard drive is used instead. Therefore, the image reading speed decreases tremendously.

Separate train set and test set data

Some random numbers are generated, and the number of which equals to that of images. 80% of the images are put into trainset, and 20% of the images are put into the test set. The random number is the same for labels and images, ensuring that they are matched.

Four randomly chosen images are shown with their normalised humidity data.

New position of two cameras

  • I have moved the normal camera to the top of the box, so that it can capture many leaves, and if one leaf moves, it does not affect the whole image. If the camera is positioned at one specific leaf as I did before, there will be no leaf in the image due to the growth and movement of the plant.
  • Also, I have fixed the magnifying camera right above one of the stems. The stem is fixed on the box by tapes, so that there will be less movement of stems than leaves.
