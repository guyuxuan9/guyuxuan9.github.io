Garbage Classification

Posted on 2020-03-23 | In Machine Learning |

(image from https://sce3a3b6c0d5pq-sb-qn.qiqiuyun.net/files/default/2019/07-16/103730af10be884644.jpg)

Recently, I took part in a competition about the garbage classification, which aims to classify the garbage into 198 categories by using image processing and CNN algorithms. The ultimate goal is to make the machine able to identify the garbage and tell the households which rubbish bin should this kind of rubbish go into. Hence, a lot of machine learning is needed to be done at first to make the machine familiar with all kinds of rubbish, and this is what the programme mainly about.

First of all, the data needs to be pre-processed. There are 80,000 images that are available to be trained. The original format of the file is JSON (JavaScript Object Notation), so the data is read into the code using json.load() method, and the return of which is a dictionary, and there are 6 keys in the dictionary, which are 'infor', 'licenses', 'images', 'type', 'annotations', 'categories' respectively.

In order to make it easier to find the name of the rubbish, names in the categories are given to a new list called names.

Then, we can randomly choose an image from the train set to have a look at it.

Note that as the index of the list names starts with 0, the index of the category_id needs to -1 to match the index.

As shown below, this is a randomly chosen umbrella.

After this, the id, label, name and filename of those rubbish can be rewritten into a new csv file called train.csv, and the process of pre-processing data is finished.

The train.csv looks like this:

Secondly, we need to train the garbage classifier. As usual, the modules are imported, but a new thing is introduced here, which are shown below. The first line aims to show the Chinese label normally, and the second line is used to show the -ve sign normally.

Then, pandas module is used to read the csv file which is created previously, and the format is DataFrame.

After reading the file, keras.preprocessing.image.ImageDataGenerator is used to load the data instead of using RAM to load the data. There are some pros and cons of each. If the data is loaded using RAM, the speed of which is very high as it directly access the memory in the computer, while if the data is loaded using keras.preprocessing.image.ImageDataGenerator, the memory will not be occupied too much. Hence, due to the fact that usually laptops do not have too much memory, generator is used here to load the data.

The first layer is the 2D convolutional network, and the size of the kernel is 5 * 5, meaning that a 5 * 5 matrix moves along each line and convolute with each element, so that the number of calculations is significantly reduced. The next layer is a 2D Max pooling one. meaning that the maximum number in a 2 * 2 matrix is chosen to represent the whole 2 * 2 matrix, so that the number of calculation will be reduced greatly although there may be a little bit loss of detail, and after which this process is repeated, followed by a flatten layer to flatten the multidimensional matrix to a 1D matrix. This is used as input to the full connection Dense layer. Hence, the summary of the model looks like this:

As shown in the table, there are 2,441,162 parameters needed to be trained, so it takes a long time to train the model.

Here, the .fit_generator() is used to train the model. steps_per_epoch = STEP_SIZE_TRAIN means that after the generator returns data for STEP_SIZE_TRAIN times, one epoch is finished. epochs means the times of iteration. After this, the model is evaluated and saved.

Then, we can use the saved model to make some predictions.

Note that the update() method inserts the specified items to the dictionary. It is like the append() method.

In the 8th line, np.expand_dims() changes the shape of the x (pixels of image) from a 122 * 122 * 3 matrix to a 1 * 122 * 3 matrix, so that it matches the format of the input of the model. Furthermore, in the 12th line, the pred has a shape of 1 * 198, because the last neuron has 198 output, and the value of which stands for the probability of being one of the 198 categories of the rubbish. Hence, the largest probability is found using argmax() method.

Finally, we can randomly choose 20 images to visualise the predicted result.

As shown in the graph, there are some incorrect prediction. For example, in the (2,3) image, it is like a ceramic, but the predicted result shows that it is a plastic cup. Therefore, some improvement of the model is needed to be done, and I will learn some improvement measures in the next week.
