nRF24 & MPU

Posted on 2020-08-31 | In car |

Today, I have learnt some functions in two modules, nRF24 and Motion Processing Unit (MPU).

nRF24 (Wireless RF Module)

The nRF24L01 is a wireless transceiver module, meaning each module can both send as well as receive data. They operate in the frequency of 2.4GHz, which falls under the ISM band and hence it is legal to use in almost all countries for engineering applications. The modules when operated efficiently can cover a distance of 100 meters (200 feet) which makes it a great choice for all wireless remote controlled projects.

  • 2.4GHz RF transceiver Module
  • Communication Protocol: SPI (Serial Peripheral Interface)

I have connected it with Arduino to receive signals from MPU that are used to transmit signals. MPU consists of gyroscope and accelerometer. Gyroscope is used to detect the rotation of the device, and accelerometer is used to detect the acceleration of the device.

Motion Processing Unit (MPU)

  • MEMS 3-aixs accelerometer and 3-axis gyroscope values combined
  • Communication : I2C protocol

(I2C supports multiple devices on the same bus without any additional select lines (work on the basis of device address). SPI requires additional signal (slave select lines) lines to manage multiple devices on the same bus. I2C is better for long-distance. SPI is better for a short distance.)

The MPU6050 is a Micro Electro-Mechanical Systems (MEMS) which consists of a 3-axis Accelerometer and 3-axis Gyroscope inside it. This helps us to measure acceleration, velocity, orientation, displacement and many other motion related parameter of a system or object. This module also has a (DMP) Digital Motion Processor inside it which is powerful enough to perform complex calculation and thus free up the work for Microcontroller

UART: (Universal Asynchronous Reception and Transmission ) –> No clock

I2C or IIC (Inter-Integrated Circuit) –> has clock, communicate through address

SPI (Serial Peripheral Interface) –> has clock, communicate by pulling down voltage level –> one more device, one more line

SS1,SS2,SS3 –> Chip Enable （片选）
