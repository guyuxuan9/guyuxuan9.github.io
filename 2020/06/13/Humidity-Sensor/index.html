Humidity Sensor

Posted on 2020-06-13 | In Dialogue with Plants |

Yesterday, I have tried to use the humidity sensor to read the humidity through Arduino and display the data using Python.

The sensor looks like the graph below:

The circuits in it looks like the graph below:

  • There is a material that is sensitive to the humidity change, so when the humidity changes, the resistance of the material changes, and therefore, the voltage output changes. In this way, the humidity can be determined.
  • Vout is the output voltage representing the humidity, and the potential divider is used to share the voltage with the humidity sensor.
  • Initially, I just apply the voltage across the humidity sensor in one direction, but I found that the data read continues to decrease. Later on, I realised that if the p.d. is applied in one direction, the interference, such as change in environment humidity, will be in one direction. By contrast, if the voltage is applied in two directions, the interference can be ‘ cancelled’.
  • Additionally, I found that there are some fluctuations in the graph being plotted, so I decided to use the Mean Filtering to make the graph look more smooth.
    $$g\left( x,y\right) =\sum _{f\in S}f\left( x,y\right)$$

Then, I use Python to plot the graph.

The final result looks like the graph below:

  • time.sleep(3) is used to delay the execution of the programme for 3 seconds, which gives time for Arduino to read the data and transfer it into my PC.
  • plt.ion() and plt.ioff() is used to plot dynamic graphs. The default mode of matplotlib is the Block Mode, so if we want to draw the dynamic graphs, we need to change it to Interactive Mode.
  • The main idea behind drawing a dynamic graph is to clean the previous graph, and draw the new graph. If this process occurs very fast, it seems to have a dynamci effect.
  • plt.pause(0.1) is used to maintain the graph just being drawn for some time, otherwise, keeping drawing the graph will make the program No response.
