Multithreading

Posted on 2020-06-24 | In Dialogue with Plants |

Yesterday, I have learnt an important concept in computer architecture called “Multithreading”. It is the ability of a CPU to provide multiple threads of execution concurrently, supported by the operating system.

< Image from Here) >

Example code

  • I learnt this concept from this example code above. func1() and func2() are all child thread, and the lines below “if name == ‘main‘: “ are the main thread.

  • func1() is used to print the current time with 1 second interval for 10 times, and func2() is used to print the current time with 2 second interval for 10 times. Therefore, func1() is runed for 10 seconds, and func2() is runed for 20 seconds given that they are run fully.

  • if __ name __ == ‘__ main __’ means if the codes is run directly, this will be executed. However, if this .py file is imported, it will not be executed.

  • .setDaemon(true) means this child thread will end as the main thread ends, and the default for this method is True.

  • t1.start() means it is the background that process the data instead of the program.

Another example of multithreading

  • Assume t is the main thread, t1,t2 are the child thread.

  • Here, t1.setDeamon(False), meaning that the main thread will end when t1 ends, so at the moment where t1 ends, t2 is not ending, but it is cut off.

  • E.g.

    t is the event that a person drives from A to B

    t1 is the event that the person get off the car

    t2 is the event that the music is on

    When the person gets off the car, the music is not necessarily stopped, but we need to switch off the music when person gets off the car.

    Multithreading in the program

    The main thread is the graphic interface that allows me to click the button to take a photo

    Two chil threads are collecting data from the humidity sensor and taking photo from the plants

    When the graphic interface is closed, it makes nonsense to still turn on the camera and humidity sensor, so the setDaemon(True) is for both humidity sensor and camera

    In conclusion, the multithreading allows the CPU to do many threads(tasks) at the same time, which is more convenient and efficient.
